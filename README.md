# Mrinal Raj Portfolio Website

This repository contains the source code for my **personal portfolio website**, built with Figma‑informed design decisions and a clean, modern front‑end stack. The site showcases my work as a **UX/UI Designer and front‑end developer**, highlighting projects, skills, and design thinking processes.

## 🎯 Purpose

- Present a **professional, visually cohesive portfolio** that reflects my design aesthetic and attention to detail.  
- Demonstrate **end‑to‑end ownership** of a real product: from concept and wireframes to responsive implementation and deployment.  
- Serve as a **living case study** for hiring managers and recruiters evaluating my design and technical abilities.

## 🧩 Key Features

- **Responsive layout** that works across mobile, tablet, and desktop.  
- **Project‑focused sections** with thumbnails, brief descriptions, and direct links to case studies or live demos.  
- Clean typographic hierarchy and **consistent color system** aligned with my personal brand.  
- Fast loading and minimal dependencies, optimized for GitHub Pages performance.

## 🛠️ Tech Stack

- **HTML, CSS, JavaScript** for structure, styling, and interactivity.  
- **Figma** for wireframing, high‑fidelity mockups, and design system documentation.  
- **GitHub Pages** for hosting and continuous deployment.  
- Optional: any custom CSS framework or utility classes (e.g., Tailwind, custom utility classes) if you used them.

## 🎨 Design Process (Use‑Case Focus)

This project was treated as a **real client‑style case study** with the following workflow:

1. **Research & Goal Setting**  
   - Defined target audience (UX hiring managers, recruiters, and design‑focused teams).  
   - Identified key sections needed: About, Skills, Projects, Contact, and optionally Resume / CV.

2. **Wireframes & Information Architecture**  
   - Created low‑fidelity wireframes to map out navigation, layout, and user flow.  
   - Ensured quick access to **case studies** and **contact information** from the first view.

3. **High‑Fidelity Mockups**  
   - Designed in Figma with a consistent color palette, typography, and spacing system.  
   - Prototyped interactions such as hover states, project cards, and navigation behavior.

4. **Development & Implementation**  
   - Coded the site with **semantic HTML** and modular CSS.  
   - Ensured **accessibility basics** (contrast, keyboard navigation, alt text on images).  
   - Optimized assets for performance (compressed images, minimal scripts).

5. **Testing & Iteration**  
   - Tested across multiple browsers and devices.  
   - Collected feedback and iterated on layout, copy, and micro‑interactions.

6. **Deployment**  
   - Deployed the site via **GitHub Pages** at `https://mrinalrajl.github.io/mrinal-raj-portfolio/`.  
   - Set up a custom domain (if applicable) for a polished URL.
